# 🧭 Evolving Documentation

This project’s CloudFormation stack is actively evolving, and the documentation evolves with it.
Rather than maintaining multiple READMEs, scattered notes, or outdated fragments, the goal is to
keep one primary README that always reflects the current state of the infrastructure.

This file exists solely to explain that philosophy once, in one place, without cluttering the
main README. The README will continue to be the single source of truth for the stack, updated
as components are added, removed, or refined.

Since this project is also part of my own process of learning and practicing Git, the README
gets updated through real commits, branches, and merges. Treating documentation as part of the
workflow helps reinforce good Git habits: small changes, clear commit messages, and a history
that actually tells the story of how the infrastructure grew.

In short: the README stays current — this file simply explains why.
